Agent Compliance Enduser Bootstrap

Öffentliche Bootstrap-Seite für Enduser-Repositories. Die Seite erstellt eine Kopie aus dem Community-Template, stellt bei Bedarf Public-Visibility für GitHub Pages her, setzt Variables und startet die ersten Workflows.

0. Warum dieser Einstieg?

Der Enduser benötigt keinen Zugriff auf ein privates Entwicklungsrepository. Die eigene Arbeitskopie wird aus einem öffentlichen, bereinigten Community-Template erzeugt und anschließend im GitHub-Account des Endusers konfiguriert.

Wichtig: GitHub Pages funktioniert bei kostenlosen GitHub-Konten nur für öffentliche Repositories. Wenn das Zielrepository private ist, stellt die Seite es vor der Pages-Aktivierung automatisch auf public um.
MOC-Grenze: Keine Exchange-Online-Berechtigungen, kein Exchange.ManageAsApp, keine Mailbox-Zugriffe und keine Exchange Online PowerShell.

3. GitHub Token

Für den vollautomatischen Browser-Flow wird ein kurzlebiger GitHub Token benötigt. Der Token wird nicht gespeichert, nicht in die URL geschrieben und nicht ins Repository committet.

Empfehlung: Token direkt nach dem Test widerrufen. Der Token muss zum Target Owner passen, z. B. ankbs.

Benötigte GitHub-Rechte

  • Administration: read/write.
  • Contents: read/write.
  • Workflows: read/write.
  • Actions: read/write.
  • Variables: read/write.
  • Pages: read/write.
