Agent Compliance Community Setup

Self-Service-Assistent für die Community Edition. Diese Seite kann lokal oder als GitHub Page laufen, Eingaben automatisch aus URL-Parametern übernehmen, GitHub Repository Variables setzen und die ersten GitHub Actions Workflows starten.

1. Überblick

Der Tester startet idealerweise mit der GitHub Page dieses Repositorys. Die Seite erkennt Owner/Repository, kann Werte aus URL-Parametern vorbefüllen und optional die GitHub Actions API nutzen, um Setup-Variablen zu speichern und Workflows zu starten.

Wichtig: Im MOC werden keine Exchange-Online-Berechtigungen, keine Mailbox-Zugriffe, kein Exchange.ManageAsApp und keine Exchange Online PowerShell verwendet.
Worker-first: Microsoft PowerShell-Module werden nicht lokal auf dem Client installiert. Requirements, Checks, Collectors und Reports laufen über GitHub Actions Runner oder Azure Cloud Shell.

Wird auf GitHub Pages automatisch aus der URL erkannt, sofern möglich.
URL-Parameter werden unterstützt, z. B. ?tenantId=...&tenantDomain=...&readerUpn=.... Dadurch kann ein Tester die Seite bereits vorausgefüllt öffnen.

3. Automatisierung über GitHub Page

Die Seite kann GitHub Repository Variables setzen und Workflows auslösen. Dafür benötigt der Tester einen kurzlebigen Fine-grained PAT für genau dieses Repository mit mindestens Variables: read/write und Actions: read/write.

Sicherheit: Der Token wird nicht gespeichert, nicht in die URL geschrieben und nicht ins Repository committet. Er wird nur im aktuellen Browser-Speicher für die API-Aufrufe genutzt. Nach dem Test sollte der Token in GitHub widerrufen werden.
Nur für automatische API-Aufrufe. Für reines Kopieren leer lassen.

6. Testablauf für Tester ohne App-Secrets

Initialtest: Der Tester kann bereits prüfen, ob Repo-Struktur, Workflows, Manifeste und Setup-Daten korrekt sind, auch wenn noch keine Entra Apps und Secrets existieren.

Automatischer Ablauf

  1. Daten erfassen oder per Setup-Link vorausfüllen.
  2. Fine-grained GitHub PAT in das Tokenfeld einfügen.
  3. Variablen setzen + Initial-Workflows starten klicken.
  4. In GitHub Actions die Artefakte prüfen.

Manueller Fallback

  1. 10 - Bootstrap Plan mit den generierten Inputs starten.
  2. 00 - Validate Worker Requirements starten. Beim ersten Lauf skip_module_install=false.
  3. 20 - Check Permissions mit skip_certificate_secret_checks=true starten.

Lokaler Schnelltest

Öffnet diese Setup-Seite lokal aus dem Repository.

Bitte zuerst Konfiguration erzeugen.

7. Was noch nicht vollständig automatisiert ist

Die GitHub-Seite automatisiert die Repository-Variablen und Workflow-Starts. Die Microsoft-Seite braucht weiterhin eine eigene Vertrauensstellung.

  • Entra App Registrations für Reader, Status Action und Billing Change erstellen.
  • API Permissions je App-Profil gemäß Manifest hinzufügen.
  • Admin Consent bewusst erteilen.
  • Zertifikate erzeugen und Public Certificate an App Registration hängen.
  • Client IDs und Zertifikatswerte als GitHub Secrets setzen.
Entra App Bootstrap ist über setup/entra-bootstrap.html verfügbar. Der Admin erhält dort einen Invoke-Befehl, der App Registrations erstellt und Admin-Consent-Links ausgibt.
